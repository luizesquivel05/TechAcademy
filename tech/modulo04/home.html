Introdução ao CSS.


-CSS (Cascading Style Sheets ou Folha de Estilo em Cascatas), descreve como os elementos HTML devem ser apresentados e define a apresentação (layout e estilo) de uma página.
-Vantagem de usar CSS: separação entre conteúdo e formato, fácil e rápido de alterar, reutilização e responsividade.
-Navegador WEB (ler o site), VSCode ou CodePen (escrever CSS).
Funciona a partir de Seletor: define quais os elementos serão afetados; Regra: define como os elementos serão afetados. Essa regras tem 1)propriedade: define a característica que será modificada; 2)valor: define como a característica será modificada.
seletor { propriedade: valor }


Aplicações do CSS.


-Sintaxe é um conjunto de regras que permite o uso da linguagem CSS: seletor { regras (propriedade: valor;) }
-3 formas de usar o css: 1)Por meio do atributo no próprio elemento; 2)Por meio da tag style no corpo do código; 3)Por meio de um arquivo externo e linkado a página;
-Seletor de elemento: pega uma tag html e aplica uma regra (a, p, h1...)
-Regras básicas: 1)Cor do texto (color: red;) 2)Sublinhado (text-decoration: underline;) 3)Cor do fundo (background-color: green;) 4)Borda (border: 1px solid black;)


Classes e IDs


Seletores são usados para identificar os elementos HTML e aplicar uma regra a ele.
Seletor elemento é uma regra geral para todos os elementos de uma mesma tag.
Seletor classe é uma regra um pouco mais específica para vários elementos parecidos. ( class - . )
Seletor id (identificador) é a regra mais específica para um elemento específico do HTML ( id - # )


Combinando selectors


Podemos usar o seletor ( * ) que é uma regra geral para todos os elementos.
Uma mesma regra pode ser geral para todos os elementos ( seletor1, seletor2 )
Podemos especializar as regras para coisas especificas ( seletor_elemento#seletor_id OU seletor_elemento.seletor_class)
Podemos filtrar as regras a partir de uma regra anterior ( regra_seletor_pai > regra_seletor_filho [caso seja filho] OU regra_seletor regra_seletordescedente [caso nao seja filho direto] )


Seletores de atributo


Para fazer comentários em CSS: /* texto */
Seletores de atributos: permite realizar a seleção a partir do valor ou da presença dos atributos.
tag [atributo] { regra }


Pseudo classes


Usado para definir um estado específico de um elemento (ex: link visitado, cursor do mouse)...
Ela dar dinamicidade e faz com que um elemento que dado tal evento (estado), tal regra deverá ser seguida.
seletor:pseudoclasse { regra }

Exemplo:
a:link { … } (não visitado)
a:visited { … } (visitado)
a:hover { ... } (mouse sobre o link)
a:active (selecionado)
input:focus (ele está no foco)
input:checked (selecionado)
input:required (ele é obrigatório)
:first-child (primeiro filho de um elemento)
:last-child (último filho do elemento)
:only-child (único filho)
:disabled (elemento desabilitado)
:empty (elemento sem filhos)


Pseudo elementos


Estados para formatar partes específicas de um elemento.
Vamos considerar pedaços do texto:

seletor::pseudoelemento { regra } Exemplo:
seletor::first-letter (primeira letra)
seletor::first-line (primeira linha)
seletor::marker (alterar o marcador)
seletor::selection (parte selecionada).


Especificades


-Os elementos podem posuir regras diferentes e contraditórias. O navegador considera o seletor mais específico, ou seja, ele usa pontuação de especifidade.

A hierarquias de especificade considera:
Estilo de linha: coloca na tag html o atributo style.
IDs: usa o seletor ID.
Classes, pseudoclasses e atributos: considera o seletor de classe.
Elementos de pseudo elemento: usa-se o seletor de elementos e pseudo-elementos.

O cálculo da especifidade varia de 0 a 1000 em que se aplica o maior valor: estilo de linha 1000; ID 100; Classe 10; Elemento 1.

Podemos considerar que o navegador deve ignorar a especifidade por meio da regra !importante, contudo, seu uso deve ser evitado.


Herança


Alguns elementos do HTML possuem características padrão que o próprio navegador fornece ao elemento, caso o mesmo não seja definido pelo CSS em sua especifdade.
Algumas propriedades são herdadas do seu descendente caso não seja especifizado (cor do texto, fonte e fundo), outras são 'resetadas' a cada nível (bordas, margem e padding).
As propriedades não herdadas podem passar a herdar pela regra inherit (seletor { border: inherit } )
Podemos resetar as apresentações padrões do navegador por intermédio da biblioteca CSS Reset , sendo assim, evita-se conflitos de apresentação.


Box model [modelo de caixa]


Esse conceito é usado quando falamos sobre layout e todos os elementos HTML são considerados uma caixa, sendo assim, algumas características que ela possui e se define os espaços entre elementos e suas marcações (margem, borda, espaço interno e conteúdo).
O conteúdo da caixa, apresenta textos e imagens, o mesmo pode ser alterado por meio de regras CSS.
O espaço interno é chamado de padding, em que a área entre conteúdo e borda da caixa e é sempre transparente
A borda é o que define os limites da caixa e é externo ao espaçamento interno e conteúdo e pode ter um tamanho, estilo e cor específica.
A margem é o expaço externo e define uma parte da distância entre a borda de 2 caixas e é sempre transparente.
As caixas podem ser oganizadas em bloco (sempre gera uma nova linha por ocupar todo espaço horizontal e respeita propriedades de altura e largura) ou em linha (as caixas não geram nova linha, seu tamanho é definido pelo conteúdo e não deve conter elementos de bloco já que ela não respeita propriedades de altura e largura).


Unidades de medida no CSS


Medidas em CSS são usados para expressar um tamanho e possuem diversas propriedades (px; cm; em; rem...), é se dado um valor seguido de uma unidade sem espaço (2cm) e elas podem ser relativas (px, em, rem, %) ou absolutas (cm, mm, in, px, pt).


Tamanho


A altura (height) e a largura (width) são definidos para elementos de blocos por meio de propriedades de: auto (navegador define), tamanho (um tamanho específico a partir de unidades de medidas), initial (a partir de tamanhos padrões no site) e inherit (herda do pai).
Podemos definir que tal elemento terá largura máxima (max-widht) ou altura máxima (max-height).
Podemos definir que tal elemento terá largura mínima (min-widht) ou altura mínima (min-height).


Margin e padding


A margem permite que a gente crie e define espaços entre elementos (bordas) por meio da propriedade margin: auto (automático) tamanho fixo ou inherit (mesmo valor do pai).

A propriedade margem pode receber 4 valores: seletor { margin: topo direita base esquerda; }
A propriedade margem pode receber 3 valores: seletor { margin: topo direita-esquerda base }
A propriedade margem pode receber 2 valores: seletor { margin: topo-base direita-esquerda }
A propriedade margem pode receber 1 valor: seletor { margin: topo-base-direita-esquerda }
Pode ser só no topo (margin-top), só na base (margin-bottom), só na esquerda (margin-left), só na direita (margin-right).

O padding cria espaços entre o conteúdo e a borda do elemento usa a propriedade padding com os valores tamanho ou inherit. Respeita mesmas regras de topo, base, direita e esquerda do margins


Bordas


A borda é definida a partir de 3 características (estilo, tamanho e cor) atribuido por: border: tamanho estilo cor .
A borda pode ser um estilo determinado pelo border-style que pode ser: dotted, dashed, double, solid, groove, none e hidden. Seu estilo pode ser topo direita base esqueda.
Podemos definir o tamanho da borda por meio da border-width em que segue unidade de medida CSS e segue o padrão de 4, 3, 2 ou 1.
Podemos definir a cor da borda por meio da propriedade border-color e também segue os padrões 4, 3, 2 ou 1.
Também segue a regra de ser pra cima (border-top), pra baixo (border-bottom), pra esquerda (border-right) ou direita (border-left).


Cores:


O sistema de cores respeita regras: nominal (definido pelo nome), RGB (definido peloi padrão RGB)HEX (definido pelo hexadecimal) e HSL (definido pelo padrão HSL)
RGB: definido pelo Red Green Blue em que 0 é a usencia plena e 255 é presença plena - rgb(255, 0, 0) [vermelho], rgb(0, 255, 0) [verde], rgb(0, 0, 255) [azul], rgb(0, 0, 0) [preto] rgb(255, 255, 255) [branco]. Após o terceiro número pode-se coloca 0 (transparente) ou 1 (não transparente)

HEX: Uma cor hexadecimal é especificada com: #RRGGBB, onde os inteiros hexadecimais RR (vermelho), GG (verde) e BB (azul) especificam os componentes da cor. #000000 [preto], #ffffff [branco], #ff0000 [vermelho], #00ff00 [verde], #0000ff [azul].

HSL: Hue, Saturation, Lightness (Matiz, Saturação e Luminosidade), isto é, Matiz define a cor em uma escala de cores, Saturação define a tonalidade de cinza, Luminosidade define a quantidade de luz.
hsl(matiz [0 a 360], saturação [0 a 100%], luminosidade [0 a 100%]) pode ter 0 (transparente) e 1 (não transparente depois.

DICAS ÚTEIS: O sistema de cores são correspondentes e opte por utiliza paletas de cores para criar uma experiência melhor para o usuário.


Background


O fundo é definido pela propriedade background e pode possuir: cor, imagem, repetição e posição.
A propriedade background-color define uma cor de fundo pode seguir padrões RGB, HEX e HSL e também ter opacidade definida.
É possível atribuir uma imagem de fundo pela background-image: url('localdaimagem')
É possível definir como uma imagem de fundo irá se repetir, pela repeat (todas as direções) repeat-y (verticalmente), repeat-x (horizontalmente) e no-repeat (não repetir).
Principalmente utilizado se a imagem de fundo não se repetir. A propriedade utilizada é background-position. Os valores para esta propriedade são: A B - [left | right | center] [top | center | bottom] x%y% inherit.
Podemos definir o quão opaco é o nosso fundo: opacity: 0 (totalmente opaco) e 1 (nada opaco).
