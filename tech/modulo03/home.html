A origem de tudo:


-Todas as vezes que você acessa uma página WEB, usamos o HTML para criá-las;
-Tim Lee na década de 1980 criou 3 elementos de página WEB: linguagem de marcação (HTML), protocolo de comunicação (HTTP) e lugar onde acessar (browser);
-Uniu 3 conceitos já existentes: internet e hypertexto;
-HTTP transita páginas HTML em distintos computadores, por meio do servidor onde ele está;
-Como funciona o site? primeiro se cria um documento que cria regras para hypertextos, depois um protocolo que liga entre computadores esse documento e, depois, um software (browser) apresenta a leitura desses padrões (regras) do hypertexto;
-A principal característica é seguir padrões (HTML e HTTP) que lhes dá uma interoperabilidade, ou seja, ela consegue rodar em todos os sistemas operacionais.


A linguagem HTML e como ela está estruturada:


-HTML (HyperText Marcage Language), é a linguagem que marca e estrutura o conteúdo que será exibido ao usuário;
-Linguagem de marcação (não é de programação) usa tags para marcar (descrever) o conteúdo dando-lhes regras específicas;
-Documento HTML é um texto demarcado por um texto com tags e regras (popularmente conhecido como página WEB);
-O documento HTML possui uma hierarquia: se cria uma tag (pai) e se coloca outra tag lá dentro (filho);
-Os browser renderiza (interpreta) o código HTML dando-lhe um formato vísivel e compreensível.


Como criar um documento HTML?


-Pode ser criado em qualquer editor de texto, desde um bloco de notas até um editor próprio [no curso usa VisualStudioCode];
-Extensões usadas: HTML snippets e HTML/CSS support;
-A estrutura mais simples do HTML tem:
   doctype: html para dizer o tipo de documento HTML;
   A tag pai HTML, diz que tudo que está ali fará parte do documento HTML;
   A tag filho head, diz o cabeçário onde se diz tudo que é necessário para apresentar o conteúdo;
   A tag filho body, diz todos os conteúdos estarão ali contidos.


Tags de abertura e fechamento.


-Tag é o que permite der uma regra, a essência da linguagem HTML;
-Tag de abertura e fechamento: se começa com o nome da tag entre < e > o conteúdo e, por fim, seu fechamento entre </ e >;
-Tag só de abertura: se tem só a tag e alguns atributos.


Tags de apresentação e tags de organização.


-Tags de apresentação: são tags que mostram algum elemento h (título), p (paragrafo), a (ancorar links) e img (colocar imagem);
-Tag de organização: serve para organizar os elementos: div (divisão), section (seção), article (artigo).


Tags e seus atributos.


-Coloca-se o nome do atributo e define-se um valor, pode ser especifico (como src) ou global (como id).


Tag title


-É uma tag que dá o titulo para sua página (na barra de navegação);
-Muito usado por ferramentas de busca para encontrar sua página do site.
<title> titulo da página </title>


Tag metacharset


-Usada para colocar acentos na página WEB.
<meta charset="UTF-8">


Tags de páragrafo, span e cabeçário


-São elementos textuais da página:
1)Tags de títulos (heading): h1 (maior) até h6 (menor).   <h1> texto </h1>

2)Tags com conteúdos: p   <p> parágrafo <p>

3)Tags de sub-divisão (texto dentro do texto): span   <span> sub texto <span>


Tag para formatação pessoal:


-Tags usadas para dar um estilo especial:
A tag strong deixa em negrito.
A tag em deixa em itálico.
A tag u deixa texto sublinhado

Tag img


-Usado para colocar imagem no texto:
<img src="" alt="">

Tag video


-Usado para incluir um video na página:
<video src="" controls> texto a ser exibido caso o navegador não encontre o vídeo. </video>

Elementos inblock e inline:


-inblock: ocupa todo o espaço do pai (ocupa toda a linha);
-inline: ocupa somente o espaço dele (mais de um elemento na mesma linha).
-A tag br quebra a linha dos elementos inline.

Hierarquia no HTML:


-HTML é feito por elementos pais, filhos, netos e descendentes.
1ª hierarquia - html
2ª hierarquia - head e body (irmãos)
Outras hierarquias - section div article aside
-Hierarquia existe em todos os elementos da página HTML.

Entendendo o DOM:


-É um objeto que compreende de forma compreensível o modelo de árvores do HTML, em tempo de execução.
-Para acessar, usamos a parte de desenvolvedor (inspecionar) o site.
-É bem parecido com HTML, porém, permite sua visão de forma mais prática, ela só muda em tempo de execução.
-O DOM existe para criar um elemento HTML que altera em tempo de execução, por meio de javascript.
-Carregamento de atualização constante e não precisa retornar no servidor pra pegar as atualizações.


Como trazer manipulações do DOM?


1)getElementById = capturar um elemento específico do DOM, window.document.getElementById para pegar o documento
1.1)Podemos colocar o getElementById dentro de uma variável e usar a função innerHTML para alterar seu conteúdo

2)getElementsByClassName e getElementsByTagName = capturar elementos múltiplas características.

Eventos DOM:


-Clique: apesar de tal clique algo aconteça;
-Dada tal mudança em caixa de texto faça isso ou aquilo;
-Como fazemos o evento?
1.Mapear o elemento (por meio de id e variável);
2.Adcionar um monitor de eventos;
2.1.Usa a função addEventListener; 2.1.1.Como primeiro parametro qual o evento é; 2.1.2.Usa-se uma segunda função para fazer com que possamos dizer o que deve acontecer.


Navegando entre elementos DOM:


-Podemos navegar entre vários elementos filhos:
função children diz os filhos do elemento
função firstElementChild diz o primeiro filho do elemento.
função parentElement diz os pais do elemento.


Tags semânticas:


-Como organizar várias seções da página;
-Se usar muita div, pode ter uma sobrecarga de desenvolvedor e díficil compreensão para interpretadores do HTML;
-Os elementos computacionais (como browser, leitor de tela e navegador de busca) tem dificuldade pra entender o site com muitas divs;
-Usamos tags containers para organizar as seções do site, para que elementos computacionais saibam o que aquele container significa;
-header (cabeçário), nav (navegação), footer (rodapé), code (código), summary (sumário), address (endereço), dfn (definir algo), article (artigo), details (dar detalhes)....

A Header


-head (tem metadados e se tem linkado coisas pra fazer o site funcionar) é dferente de header (tem a função) de representar o cabeçário.


A nav


-A nav é usada para definir um conjunto de links de navegação, links que nos levam para um lugar.


A main


Main - serve para especificar a parte principal (NÚCLEO) da página.


A Footer e outras tags semânticas


Footer - Representa o rodapé do documento e da seção, tem mapa do site e informações de quem fez.
Figure - cria uma imagem e permite colocar imagem e legenda.
Code - coloca uma região em código.
Abbr - Abreviatura pra detalhar algo.
Details - para detalhar um dado.
Aside - marginal a uma main, explicar ou aprimorar ao conteúdo principal.
Address - colocar contato e endereço dentro desse conteúdo.
Article - usar o artigo do site.
Dfn - indexar e definir o conteúdo do seu site.
Dialog - cria caixa de dialógo do seu html.
Section - delimitar as seçõe do seu site.

Metadados:


-Dados que explicam dados, ou seja, informações auxiliares em compartilhamento de dados;
-Inseridos na tag head (pois é o que dá um valor externo ao site);
-Title: dá um titulo usado na barra de favoritos, usado por motores de busca.
-Meta charset: o incode de carregamento diz padrão de tradução;


Style:


-vai dentro da tag head que tem padrões básicos do css para a página, ou seja, estilizar os elementos padrões.


Script:


-vai dentro da tag head que tem padrões básicos do js para a página, ou seja, dar inteligencia aos elementos padrões.


Base:


-Vai dentro da tag head e serve para organizar mesmo site do mesmo domínio. Auxilia endereço base pra ser concatenado por padrão.


Metatags:


Quais atributos posso usar na tag meta? Para ser aprimorada pelo motor de busca...
O que o motor de busca lê? Os metadados!
meta: name e content -> name: "description" (a descrição); -> name: "keywords" (palavras chaves); -> name: "author" (quem fez); -> name: "robots" (index [indexar a página], noindex [não indexar], follow [seguir], nofollow [não seguir], noarchive [não deixar cache] e nosnippet [não exibir descrição]).


Metatags sociais:


-Aquelas usadas para deixar rede social e site compatíveis. auxiliam na descrição de compartilhamento em redes socias (como insta, twitter, facebook).


FavIcon:


O icone que fica na barra de navegação.
<link rel="shortcut icon" type="image/x-icon" href="onde está?">


Hyperlinks:


-São as formas como podemos conectar sites, usados pela tag a o atributo target (_blank) para dizer o alvo (local) onde vai abrir.


Âncoras:


-São hiperlinks que vão para a mesma parte da mesma página, que é feito pela ID;
Para ir pra outra página numa parte específica, para isso, usamos o atributo href com o hyperlink com #id.


Links especializados:


Criar links que dirige pra lugares específicos, por meio, da tag a, atributo href ("mailto: email da pessoa" ou "tel: o número da pessoa.")


URL:


Localizador Uniforme de Recursos -> padrão pro navegador encontrar uma página WEB.


Navegando por diretórios:


Caminhos relativos (a partir de onde o arquivo está) e caminhos absolutos (desde a raíz do projeto). Pode ter navegações internas (mesmo projeto) ou externa (fora da página). A navegação dentro da mesma aplicação pode ser feita por: . (mesmo nível) ou .. (nível anterior)/nome do arquivo.


Tabelas:


É uma planilha formada linha e coluna.
Envolvida pela table, a sua linha é feita por tr e cada célula será feita por td


Bordas:


Podemos usar o atributo border na tag table.


Tabelas complexas:


são tabelas em que uma célula ocupa um espaço maior que a coluna deveria ocupar.
Podemos usar o atributo com colspam para mesclar um conjunto específico de colunas.
Podemos usar o atributo rowspam para mesclar um conjunto específico de linhas.


Tags de tabela:

-Podemos criar cabeçário, corpo e rodapé da tabela, por meio de tags semânticas.
Cabeçário é feito por thead
Corpo é feito por tbody
Rodapé é feitto por tfoot
A tag caption pode ser usado logo após abertura da tag table e criar uma legenda
